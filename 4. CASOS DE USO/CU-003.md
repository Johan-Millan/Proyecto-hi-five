# CU-003 – Actualizar Información de Participante

## Información General
- **ID:** CU-003
- **Nombre:** Actualizar Información de Participante
- **Módulo:** Gestión de Usuarios y Roles
- **Prioridad:** MUST
- **Complejidad:** Media
- **Estado:** En Revisión
## Actores
- **Actor Principal:** Personal Administrativo
- **Actores Secundarios:** Sistema (auditoría y validaciones automáticas)
## Precondiciones
- El participante debe existir en el sistema. 
- El usuario debe haber iniciado sesión con permisos adecuados. 
## Postcondiciones
- La información del participante queda actualizada y almacenada correctamente.
- El sistema registra la acción en la bitácora de auditoría. 
## Flujo Principal (Camino Feliz)
1. El actor accede al módulo de gestión de participantes. 
2. El actor ingresa el número de documento para buscar al participante. 
3. El sistema muestra el registro encontrado. 
4. El actor selecciona la opción “Editar información”. 
5. El actor modifica los campos requeridos. 
6. El sistema valida el formato y consistencia de los datos ingresados. 
7. El sistema guarda los cambios realizados.
8. El sistema registra la actualización en el log de auditoría.
9. El sistema confirma que la información ha sido actualizada exitosamente.
## Flujos Alternativos
### A1 – Participante no encontrado
- El sistema no encuentra coincidencias.
- El sistema notifica al actor.
- El caso de uso finaliza.
### A2 – No se modifica ningún dato
- El actor no cambia ningún campo.
- El sistema permite cancelar sin guardar cambios.
## Flujos de Excepción
### E1 – Error de validación en los datos
- El sistema detecta campos con errores 
- Muestra mensaje indicando campos incorrectos.
- El actor corrige y retorna al paso 5.
### E2 – Fallo al guardar datos
- Ocurre un error interno en el sistema o en la base de datos.
- El sistema informa el error y sugiere reintentar.
- El caso de uso termina sin cambios.
## Reglas de Negocio
- RN-05: Solo usuarios autorizados pueden modificar información de participantes. 
- RN-09: Toda modificación debe quedar registrada en el sistema de auditoría. 
- RN-10: No se permiten datos duplicados . 
## Requisitos No Funcionales
- RNF-001.1: Interfaz intuitiva que facilite edición sin capacitación extensa. 
- RNF-001.3: Operable con teclado y ratón. 
- RNF-002.3: Búsqueda extremadamente rápida por documento. 
- RNF-003.3: Protección de datos sensibles. 
- RNF-003.4: Registro de auditoría obligatorio. 
## Interfaces de Usuario
- **Pantalla:** “Gestión de Participantes – Buscar y Editar”.
- Formularios de edición con validaciones de formato.
