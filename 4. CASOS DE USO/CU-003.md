CU-003: ELIMINAR USUARIO
INFORMACIÓN GENERAL

ID: CU-003

Nombre: Permite al personal administrativo eliminar un participante del sistema, asegurando que su información no esté ligada a procesos activos antes de proceder.

Módulo: Gestión de Usuarios y Roles – Usuarios

Prioridad: SHOULD

Complejidad: Media

Estado: Aprobado

Actores

Actor Principal: Personal Administrativo

Actores Secundarios: Sistema

Precondiciones

El usuario debe estar autenticado como personal administrativo.

El participante debe existir previamente en la base de datos.

El participante no debe estar asociado a procesos activos (capacitaciones, certificaciones pendientes, trámites en curso).

Postcondiciones

El participante queda eliminado del sistema.

Se registra en auditoría la acción de eliminación.

Flujo Principal (Camino Feliz)

El actor ingresa al módulo de gestión de participantes.

El sistema muestra la lista o buscador de participantes.

El actor selecciona un participante existente.

El sistema muestra los datos del participante junto con la opción “Eliminar”.

El actor confirma que desea eliminar al participante.

El sistema valida que el participante no esté asociado a procesos activos.

El sistema elimina el registro del participante.

El sistema confirma la eliminación exitosa.

Flujos Alternativos

A1 – Cancelación de eliminación

En el paso 5, el actor cancela la operación → El proceso termina sin cambios.

Flujos de Excepción

E1 – Participante no encontrado

3a. El sistema notifica que el participante no existe.

El flujo termina.

E2 – Participante con procesos activos

6a. El sistema detecta que el participante está asociado a trámites o capacitaciones en curso.

6b. El sistema bloquea la eliminación y muestra un mensaje informativo.

El flujo termina.

Reglas de Negocio

RN-001: Toda eliminación debe asegurar que no existan dependencias activas.

RN-003: Toda eliminación debe quedar registrada en auditoría con usuario, fecha y hora.

Requisitos No Funcionales

RNF-001.1: La interfaz debe ser clara y sencilla en la operación de eliminación.

RNF-001.3: Debe permitir interacción con teclado y ratón.

RNF-001.4: Debe permitir ajustar el tamaño de lectura.

RNF-002.2: La validación de dependencias debe ejecutarse con tiempos de respuesta aceptables.

RNF-002.5: El sistema debe soportar alto volumen de participantes y validaciones.

RNF-003.3: Debe asegurar la protección de datos personales.

RNF-003.4: Debe registrar en auditoría la acción realizada.

RNF-005.1: Debe ejecutarse en entorno web.

Interfaces de Usuario

Pantalla: Detalle del Participante

Elementos:

Botón “Eliminar participante”

Confirmación modal antes de eliminar

Alertas de validación de dependencias
