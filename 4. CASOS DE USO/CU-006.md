CU-006: EJECUTAR CIERRE MENSUAL
INFORMACIÓN GENERAL

ID: CU-006

Nombre: Permite ejecutar automáticamente el cierre mensual y generar certificados de participantes aprobados.

Módulo: Reportes y Cumplimientos – Cierre Mensual

Prioridad: MUST

Complejidad: Alta

Estado: Aprobado

Actores

Actor Principal: Sistema (automático)

Actores Secundarios: Personal Administrativo

Precondiciones

Debe haberse alcanzado la fecha de fin de mes.

Los registros de cursos aprobados deben estar completos.

El sistema debe tener acceso a la base de datos de participantes e instructores.

Postcondiciones

Se generan los certificados correspondientes a participantes aprobados.

Los certificados quedan almacenados en el repositorio.

Se notifica al personal administrativo la finalización del proceso.

Se registra el proceso en el historial de auditoría.

Flujo Principal (Camino Feliz)

El sistema detecta que se ha alcanzado la fecha de cierre mensual.

El sistema identifica los participantes que aprobaron cursos durante el mes.

El sistema genera automáticamente los certificados correspondientes.

El sistema almacena los certificados en el repositorio digital.

El sistema marca los participantes como “Certificado disponible”.

El sistema envía notificación al personal administrativo indicando que el proceso ha concluido.

El sistema registra toda la operación en auditoría.

Flujos Alternativos

A1 – Cierre manual

El personal administrativo ejecuta manualmente el proceso antes de la fecha.

El sistema continúa desde el paso 2 del flujo principal.

Flujos de Excepción

E1 – Datos incompletos en aprobaciones

2a. El sistema detecta registros incompletos.

2b. Notifica al personal administrativo para corregirlos antes de continuar.

E2 – Error al generar un certificado

3a. El sistema identifica fallos en la generación de un certificado.

3b. Registra el error y continúa con los restantes.

3c. Notifica al personal administrativo al finalizar.

Reglas de Negocio

RN-008: Solo los participantes con estado “Aprobado” pueden recibir certificado.

RN-009: Todo certificado generado debe almacenarse en el repositorio oficial.

Requisitos No Funcionales

RNF-002.1: Debe ejecutarse sin afectar el rendimiento general.

RNF-002.2: Operaciones críticas deben responder en tiempo aceptable.

RNF-002.5: Soporte para gran volumen de datos mensuales.

RNF-003.4: Debe registrarse todo el proceso para auditoría.

RNF-004.1: Debe garantizar respaldo del proceso.

RNF-005.1: Debe ejecutarse como servicio web automatizado.

RNF-006.2: La documentación del proceso debe mantenerse actualizada.
