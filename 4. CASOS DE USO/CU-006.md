# Información General

- **ID:** CU-006
- **Nombre:** Registrar Incidente
- **Descripción breve:** Permite registrar incidentes ocurridos en el centro de entrenamiento incluyendo detalles, responsables y evidencia.
- **Módulo:** Reportes y Cumplimiento
- **Prioridad:** MUST
- **Complejidad:** Media
- **Estado:** En Revisión
## Actores
- **Actor Principal:** Personal Administrativo
- **Actores Secundarios:** Sistema (validación, auditoría)
## Precondiciones
- El usuario debe estar autenticado y autorizado para registrar incidentes. 
- El módulo de incidentes debe estar disponible. 
## Postcondiciones
- El incidente queda registrado con un ID único. 
- La acción queda registrada en el sistema de auditoría. 
## Flujo Principal (Camino Feliz)
1. El actor accede al módulo “Registro de Incidentes”. 
2. El sistema muestra un formulario para ingresar datos del incidente. 
3. El actor ingresa la información requerida: fecha, responsable, descripción y evidencia. 
4. El sistema valida los datos ingresados.
5. El sistema registra el incidente y genera un ID único. 
6. El sistema almacena la evidencia asociada. 
7. El sistema registra la acción en el módulo de auditoría. 
8. El sistema confirma el registro exitoso. 
## Flujos Alternativos
### A1 – Evidencia no cargada
- El actor decide registrar el incidente sin evidencia.
- El sistema permite continuar indicando que la evidencia puede agregarse después.
### A2 – Información incompleta pero no crítica
- El sistema detecta campos opcionales vacíos.
- El sistema advierte pero permite continuar.
## Flujos de Excepción
### E1 – Error en los datos ingresados
- El sistema detecta campos obligatorios mal diligenciados.
- Notifica el error y solicita corrección.
- El actor regresa al paso 3.
### E2 – Falla al guardar el incidente
- El sistema presenta un error al intentar guardar.
- Notifica el problema y solicita reintentar.
