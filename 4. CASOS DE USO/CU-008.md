# CU-008 – Generar Certificado Individual

## Información General
- **ID:** CU-008
- **Nombre:** Generar Certificado Individual
- **Descripción breve:** Permite al participante buscar y obtener su certificado de alturas tras aprobar un curso, asegurando acceso seguro y verificación de identidad.
- **Módulo:** Gestión de Usuarios y Roles – Participantes
- **Prioridad:** MUST
- **Complejidad:** Media
- **Estado:** En Revisión
## Actores
- **Actor Principal:** Participante
- **Actores Secundarios:** Sistema
## Precondiciones
- El participante debe haber completado y aprobado el curso.
- El participante debe estar autenticado en el sistema. 
## Postcondiciones
- El certificado queda disponible para descarga o impresión.
- Se registra la acción del participante en la auditoría del sistema.
## Flujo Principal (Camino Feliz)
1. El participante accede al módulo “Certificados”.
2. Ingresa su número de documento y credenciales de autenticación.
3. El sistema valida la identidad del participante.
4. El sistema verifica que el participante haya aprobado el curso.
5. El sistema muestra el certificado disponible para descarga o impresión.
6. El participante imprime o descarga el certificado.
7. El sistema registra la acción en el historial de auditoría.
## Flujos Alternativos
### A1 – Solicitar correo del certificado:
- El participante puede elegir recibir el certificado en su correo electrónico.
- El sistema envía el certificado al correo registrado.
## Flujos de Excepción
### E1 – Participante no aprobado:
- El sistema indica que no es posible generar el certificado.
- El flujo termina. 
### E2 – Error de autenticación:
- El sistema notifica fallo de credenciales y solicita reintento.
- Después de 3 intentos fallidos, se bloquea temporalmente el acceso. 
## Reglas de Negocio
- RN-14: Solo participantes que hayan aprobado pueden generar certificados.
- RN-15: Cada certificado generado queda registrado para auditoría.
## Requisitos No Funcionales
- RNF-001.1: La interfaz debe ser clara y sencilla para el participante.
- RNF-002.2: La generación y descarga del certificado debe ser rápida.
- RNF-003.3: Proteger datos personales y resultados del participante.
- RNF-003.4: Registrar acción en auditoría.
## Interfaces de Usuario
- **Pantalla:** “Certificados” para ingreso de datos y autenticación.
- Vista previa del certificado antes de imprimir.
- Botones de “Descargar” y “Enviar por correo”.
## Referencias
- RF-002.4: Permite a los participantes buscar e imprimir su certificado.
- HU-008: Impresión de Certificados por Participantes.
- RNF-001.1, RNF-001.2, RNF-001.3: Usabilidad y accesibilidad.
- RNF-003.3: Protección de datos personales.
