CU-010 – Generar Informes de Auditoría
Información General

ID: CU-010

Nombre: Generar Informes de Auditoría

Descripción breve: Permite al personal administrativo o auditores generar informes exhaustivos sobre registros, cursos, participantes e instructores, según criterios específicos.

Módulo: Reportes y Cumplimiento

Prioridad: MUST

Complejidad: Alta

Estado: En Revisión

Actores

Actor Principal: Personal administrativo / Auditor

Actores Secundarios: Sistema

Precondiciones

El usuario debe estar autenticado y tener permisos para generar informes de auditoría. (RNF-003.1)

Los registros de participantes, instructores y cursos deben estar actualizados y completos. (RF-002.3)

Postcondiciones

El informe de auditoría queda generado y disponible para descarga o consulta.

La acción queda registrada en el historial de auditoría del sistema. (RNF-003.4)

Flujo Principal (Camino Feliz)

El usuario accede al módulo “Informes de Auditoría”.

Selecciona los criterios de filtrado: fechas, tipo de registros, participantes, instructores, incidentes, etc.

El sistema recopila los datos correspondientes según los criterios seleccionados.

El sistema genera el informe en formato PDF o Excel.

El sistema permite al usuario visualizar, descargar o enviar el informe por correo.

La operación queda registrada en el historial de auditoría.

Flujos Alternativos

A1 – Guardar criterios de búsqueda:

El usuario puede guardar criterios de búsqueda para futuros informes.

El sistema almacena los criterios asociados al usuario.

A2 – Envío automático por correo:

El usuario solicita enviar el informe generado a un correo corporativo.

El sistema envía el informe y registra la acción.

Flujos de Excepción

E1 – Error al recopilar datos:

El sistema notifica un error y solicita reintentar la operación.

La incidencia queda registrada en auditoría. (RNF-003.4)

E2 – Usuario no autorizado:

El sistema bloquea la operación y muestra un mensaje de acceso denegado. (RNF-003.1)

Reglas de Negocio

RN-18: Solo usuarios con permisos administrativos o auditores pueden generar informes de auditoría. (RF-002.3)

RN-19: Todos los informes generados deben registrarse en el historial de auditoría. (RNF-003.4)

Requisitos No Funcionales

RNF-001.1: Interfaz clara y simple para seleccionar filtros y generar el informe.

RNF-002.2: Generación de informes en tiempo de respuesta aceptable.

RNF-003.3: Protección de datos sensibles de participantes, instructores y cursos.

RNF-003.4: Registro automático de la acción en auditoría.

Interfaces de Usuario

Pantalla de selección de criterios y filtros.

Vista previa del informe generado.

Botones “Descargar” y “Enviar por correo”.

Referencias

RF-002.3: Generación de informes de auditoría y cumplimiento.

RNF-003.1: Autenticación y control de acceso.

RNF-003.3: Protección de datos sensibles.

RNF-003.4: Registro de auditoría.
