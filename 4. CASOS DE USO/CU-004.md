CU-004 – Registrar Instructor
Información General

ID: CU-004

Nombre: Registrar Instructor

Descripción breve: Permite registrar la información completa de un instructor, incluyendo credenciales y licencias vigentes.

Módulo: Gestión de Usuarios y Roles

Prioridad: MUST

Complejidad: Media

Estado: En Revisión

Actores

Actor Principal: Personal Administrativo

Actores Secundarios: Sistema (validación, auditoría)

Precondiciones

El usuario debe tener sesión iniciada con permisos para gestionar instructores. (REF-03)

No debe existir otro instructor registrado con la misma identificación. (REF-03)

Postcondiciones

El instructor queda registrado exitosamente en el sistema. (REF-01)

El sistema registra la creación del instructor en el log de auditoría. (REF-03)

Flujo Principal (Camino Feliz)

El actor accede al módulo “Gestión de Instructores”. (REF-01)

Selecciona la opción “Registrar Instructor”. (REF-01)

El sistema muestra el formulario de registro. (REF-04)

El actor ingresa los datos personales del instructor. (REF-01)

El actor ingresa credenciales, certificaciones y licencias vigentes. (REF-01)

El sistema valida que no exista un instructor con el mismo número de documento. (REF-03)

El sistema verifica el formato y consistencia de los datos. (REF-04)

El sistema guarda el nuevo instructor en la base de datos. (REF-01)

El sistema registra la acción en la bitácora de auditoría. (REF-03)

El sistema muestra un mensaje confirmando el registro exitoso. (REF-01)

Flujos Alternativos
A1 – Instructor ya registrado

6a. El sistema detecta que ya existe un instructor con ese documento.

6b. Informa al usuario que el registro no se puede duplicar.

6c. El actor decide corregir o cancelar.
(REF-03 + REF-05)

Flujos de Excepción
E1 – Error en validación de datos

7a. El sistema detecta datos inválidos o incompletos.

7b. Muestra los campos a corregir.

7c. El actor regresa al paso 4.
(REF-04)

E2 – Fallo al guardar el registro

8a. Ocurre un error interno o desconexión.

8b. El sistema cancela la operación y notifica al actor.
(REF-05)

Reglas de Negocio

RN-03: Gestión completa de instructores con información detallada. (REF-01)

RN-10: No se permiten registros duplicados por documento. (REF-03)

RN-09: Toda creación/modificación debe quedar registrada. (REF-03)

Requisitos No Funcionales

RNF-001.1: Interfaz simple e intuitiva. (REF-04)

RNF-001.3: Accesible con teclado y ratón. (REF-04)

RNF-002.4: El registro debe realizarse de forma eficiente. (REF-04)

RNF-003.3: Protección de datos personales sensibles. (REF-04)

RNF-003.4: Auditoría obligatoria. (REF-04)

Interfaces de Usuario

Controles accesibles para ingresar datos personales y de certificaciones. (REF-04)
