# CU-004 – Registrar Instructor

## Información General
- **ID:** CU-004
- **Nombre:** Registrar Instructor
- **Descripción breve:** Permite registrar la información completa de un instructor, incluyendo credenciales y licencias vigentes.
- **Módulo:** Gestión de Usuarios y Roles
- **Prioridad:** MUST
- **Complejidad:** Media
- **Estado:** En Revisión
## Actores
- **Actor Principal:** Personal Administrativo
- **Actores Secundarios:** Sistema (validación, auditoría)
## Precondiciones
- El usuario debe tener sesión iniciada con permisos para gestionar instructores.
- No debe existir otro instructor registrado con la misma identificación.
## Postcondiciones
- El instructor queda registrado exitosamente en el sistema.
- El sistema registra la creación del instructor en el log de auditoría.
## Flujo Principal (Camino Feliz)
1. El actor accede al módulo “Gestión de Instructores”.
2. Selecciona la opción “Registrar Instructor”. 
3. El sistema muestra el formulario de registro. 
4. El actor ingresa los datos personales del instructor. 
5. El actor ingresa credenciales, certificaciones y licencias vigentes. 
6. El sistema valida que no exista un instructor con el mismo número de documento. 
7. El sistema verifica el formato y consistencia de los datos. 
8. El sistema guarda el nuevo instructor en la base de datos.
9. El sistema registra la acción en la bitácora de auditoría.
10. El sistema muestra un mensaje confirmando el registro exitoso. 
## Flujos Alternativos
### A1 – Instructor ya registrado
- El sistema detecta que ya existe un instructor con ese documento.
- Informa al usuario que el registro no se puede duplicar.
- El actor decide corregir o cancelar.
## Flujos de Excepción
### E1 – Error en validación de datos
- El sistema detecta datos inválidos o incompletos.
- Muestra los campos a corregir.
- El actor regresa al paso 4.
### E2 – Fallo al guardar el registro
- Ocurre un error interno o desconexión.
- El sistema cancela la operación y notifica al actor.
## Reglas de Negocio
- RN-03: Gestión completa de instructores con información detallada. 
- RN-10: No se permiten registros duplicados por documento. 
- RN-09: Toda creación/modificación debe quedar registrada. 
## Requisitos No Funcionales
- RNF-001.1: Interfaz simple e intuitiva.
- RNF-001.3: Accesible con teclado y ratón.
- RNF-002.4: El registro debe realizarse de forma eficiente. 
- RNF-003.3: Protección de datos personales sensibles.
- RNF-003.4: Auditoría obligatoria. 
## Interfaces de Usuario
- **Pantalla:** “Registrar Instructor – Formulario”. 
- Controles accesibles para ingresar datos personales y de certificaciones. 
## Referencias
- RF-001.3: Registro y gestión de información detallada de instructores.
- HU-003: Registrar y gestionar la información completa de los instructores.
