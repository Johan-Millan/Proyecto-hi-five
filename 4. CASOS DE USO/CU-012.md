# CU-012 – Autenticación y Gestión de Acceso

## Información General
- **ID:** CU-012
- **Nombre:** Autenticación y Gestión de Acceso
- **Descripción breve:** Permite a todos los usuarios iniciar sesión y acceder al sistema según sus roles, incluyendo validación de contraseña y políticas de seguridad.
- **Módulo:** Gestión de Usuarios y Roles
- **Prioridad:** MUST
- **Complejidad:** Alta
- **Estado:** En Revisión
## Actores
- **Actor Principal:** Todos los usuarios
- **Actores Secundarios:** Sistema
## Precondiciones
- El usuario debe contar con credenciales válidas. 
## Postcondiciones
- El usuario queda autenticado con el nivel de acceso correspondiente.
- Acciones de inicio de sesión quedan registradas para auditoría. 
## Flujo Principal (Camino Feliz)
1. El usuario ingresa sus credenciales .
2. El sistema valida la autenticidad de las credenciales.
3. El sistema verifica el rol y permisos asociados.
4. Se concede acceso al usuario al módulo correspondiente.
5. La acción queda registrada en el historial de auditoría.
## Flujos Alternativos
### A1 – Recuperación de contraseña:
- El usuario puede solicitar restablecer su contraseña según políticas de seguridad. 
### A2 – Cambio de contraseña obligatoria:
- Si la contraseña ha caducado, el sistema solicita actualizarla antes de permitir acceso.
## Flujos de Excepción
### E1 – Credenciales incorrectas:
- El sistema deniega el acceso y muestra mensaje de error.
### E2 – Usuario bloqueado:
- El sistema impide el acceso temporalmente tras intentos fallidos consecutivos.
## Reglas de Negocio
- RN-22: El acceso al sistema debe estar basado en roles y permisos definidos.
- RN-23: Las contraseñas deben cumplir políticas de complejidad y caducidad.
- RN-24: Todas las acciones de inicio de sesión deben quedar registradas.
## Requisitos No Funcionales
- RNF-001.1: Interfaz clara y sencilla para el inicio de sesión.
- RNF-002.2: Validación de credenciales rápida y confiable.
- RNF-003.1: Implementación de dos modelos de autenticación.
- RNF-003.2: Gestión segura de contraseñas y recuperación.
- RNF-003.4: Registro de auditoría de accesos.
## Interfaces de Usuario
- **Pantalla:** de inicio de sesión.
- Formulario de recuperación de contraseña.
- Alertas de validación de credenciales.
## Referencias
- RNF-003.1: Implementación de dos modelos de acceso para autenticación.
- RNF-003.2: Gestión de contraseñas con políticas de seguridad.
