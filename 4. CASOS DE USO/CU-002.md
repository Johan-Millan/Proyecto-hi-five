# CU-002: BUSCAR PARTICIPANTE
## INFORMACION GENERAL
**ID:** CU-002
**Nombre:** Permite buscar y visualizar la información de un participante mediante su documento o nombre, para consulta, validación o actualización.
**Módulo:** Gestión de Usuarios y Roles – Usuarios
**Prioridad:** MUST
**Complejidad:** Baja
**Estado:** Aprobado
## Actores
**Actor Principal:** Personal Administrativo / Participante
**Actores Secundarios:** Sistema
## Precondiciones
- El usuario debe estar autenticado.
- Debe existir al menos un participante registrado.
## Postcondiciones
- El sistema muestra la información del participante encontrado.
- Los datos quedan listos para consulta, impresión o modificación.
## Flujo Principal (Camino Feliz)
1. El actor accede al módulo de búsqueda de participantes.
2. El sistema muestra el campo de búsqueda (documento o nombre).
3. El actor ingresa un criterio de búsqueda.
4. El sistema ejecuta una búsqueda rápida en la base de datos.
5. El sistema muestra los resultados coincidentes.
6. El actor selecciona un participante.
7. El sistema despliega la información completa del participante.
## Flujos Alternativos
### A1 – Búsqueda por nombre
- El actor ingresa nombres o apellidos.
- El sistema muestra una lista de coincidencias parciales.
### A2 – Búsqueda por documento incompleto
- El actor ingresa parte del número de documento.
- El sistema muestra coincidencias aproximadas.
## Flujos de Excepción
### E1 – Sin resultados
- El sistema no encuentra coincidencias.
- Muestra mensaje “No se encontraron participantes”.
### E2 – Error de formato
- Si el documento contiene caracteres inválidos, el sistema solicita corrección.
## Reglas de Negocio
- RN-001: La búsqueda debe operar únicamente sobre registros válidos.
- RN-002: No se deben mostrar datos de participantes inactivos o eliminados (si la política lo exige).
## Requisitos No Funcionales
- RNF-001.1: La búsqueda debe ser intuitiva y sencilla.
- RNF-001.2: Debe incluir iconografía y diseño accesible.
- RNF-001.3: Debe ser operable con teclado y ratón.
- RNF-002.1: Debe soportar múltiples usuarios buscando simultáneamente.
- RNF-002.2: Debe responder en tiempo aceptable.
- RNF-002.3: La búsqueda por documento debe ser extremadamente rápida.
- RNF-002.5: Debe funcionar con grandes volúmenes de datos.
- RNF-003.3: La información mostrada debe ser protegida como dato sensible.
- RNF-003.4: Registrar la acción de consulta del usuario.
- RNF-005.1: Debe operar como aplicación web.
## Interfaces de Usuario
**Pantalla:** Búsqueda de Participantes
**Elementos:**
- Campo de búsqueda
- Botón “Buscar”
- Tabla de resultados
- Alerta de “sin resultados”
