CU-007 – Ejecutar Cierre Mensual
Información General

ID: CU-007

Nombre: Ejecutar Cierre Mensual

Descripción breve: Permite ejecutar el proceso de cierre mensual que consolida información operativa, genera reportes automáticos y bloquea modificaciones posteriores al periodo.

Módulo: Administración / Contabilidad Operativa

Prioridad: MUST

Complejidad: Alta

Estado: En Revisión

Actores

Actor Principal: Administrador del Sistema

Actores Secundarios: Sistema (procesos automáticos), Módulo de Reportes

Precondiciones

El usuario debe tener permisos de administrador. (REF-03)

Toda la información del mes debe estar registrada y validada. (REF-01)

El sistema debe estar en modo operativo normal (sin cierres en ejecución). (REF-01)

Postcondiciones

El mes queda cerrado y bloqueado para modificaciones. (REF-01)

Se generan los reportes automáticos programados. (REF-02)

Se registra la acción en la auditoría del sistema. (REF-03)

Flujo Principal (Camino Feliz)

El actor accede a la opción “Ejecutar Cierre Mensual”. (REF-01)

El sistema muestra el estado del mes actual y advierte que el proceso es irreversible. (REF-02)

El actor confirma que desea ejecutar el cierre. (REF-02)

El sistema valida que no existan procesos pendientes ni datos incompletos. (REF-04)

El sistema bloquea el periodo para evitar nuevos cambios. (REF-01)

El sistema ejecuta los procesos automáticos de consolidación. (REF-01)

El sistema genera automáticamente los reportes del cierre mensual. (REF-02)

El sistema registra el evento de cierre en auditoría. (REF-03)

El sistema notifica el cierre exitoso al actor. (REF-02)

Flujos Alternativos
A1 – Cierre programado

1a. El actor selecciona la opción “Programar cierre automático”.

1b. El sistema solicita fecha y hora.

1c. El actor confirma.

1d. El sistema agenda el proceso.
(REF-02)

Flujos de Excepción
E1 – Datos incompletos

4a. El sistema detecta información faltante (clases sin cerrar, reportes pendientes, registros vacíos).

4b. Presenta una lista de pendientes.

4c. El actor debe corregir antes de continuar.
(REF-04)

E2 – Error durante la consolidación

6a. El sistema encuentra un error técnico en el procesamiento.

6b. Detiene el proceso y revierte los cambios.

6c. Notifica al actor y registra el error en auditoría.
(REF-05)

E3 – Falla en la generación de reportes

7a. El sistema no puede generar uno o más reportes automáticos.

7b. Continúa con el cierre pero marca los reportes fallidos para revisión manual.
(REF-05)

Reglas de Negocio

RN-11: El cierre mensual es irreversible para evitar inconsistencias. (REF-01)

RN-12: Solo perfiles administrativos pueden ejecutar cierres contables u operativos. (REF-03)

RN-13: Todas las acciones críticas deben quedar en auditoría. (REF-03)

Requisitos No Funcionales

RNF-001.1: Interfaz clara y con alertas de riesgos en acciones críticas. (REF-04)

RNF-002.1: Manejo eficiente de procesos pesados (consolidación). (REF-04)

RNF-003.3: Protección de datos sensibles del periodo. (REF-04)

RNF-003.4: Registro obligatorio en auditoría. (REF-04)

Interfaces de Usuario

Diálogo de confirmación de cierre irreversible.

Pantalla de progreso del proceso.

Resumen final del cierre.
