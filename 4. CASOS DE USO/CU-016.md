# CU-016 – Recuperación de Datos

## Información General
- **ID:** CU-016
- **Nombre:** Recuperación de Datos
- **Descripción breve:** Permite restaurar los datos desde respaldos para garantizar la continuidad operativa del sistema en caso de fallos o pérdida de información.
- **Módulo:** Gestión de Usuarios y Roles / Fiabilidad
- **Prioridad:** MUST
- **Complejidad:** Media
- **Estado:** En Revisión
## Actores
- **Actor Principal:** Administrador del sistema
- **Actores Secundarios:** Sistema
## Precondiciones
- Debe existir un respaldo válido y accesible.
- El sistema debe estar operativo y con permisos de restauración.
## Postcondiciones
- Los datos son restaurados correctamente.
- El sistema vuelve a un estado funcional previo al fallo.
## Flujo Principal (Camino Feliz)
1. El administrador accede al módulo de recuperación de datos.
2. Selecciona el respaldo a restaurar (fecha y tipo: completo o incremental).
3. El sistema valida la integridad del respaldo seleccionado.
4. Se inicia la restauración de los datos al sistema.
5. El sistema confirma que la recuperación se realizó correctamente.
6. El administrador verifica que los datos estén completos y operativos.
## Flujos Alternativos
### A1 – Recuperación parcial:
- El administrador selecciona solo ciertos módulos o tablas para restaurar.
- El sistema realiza la recuperación parcial manteniendo el resto de los datos intactos.
## Flujos de Excepción
### E1 – Respaldo corrupto o inaccesible:
- El sistema notifica el error al administrador.
- Se solicita seleccionar otro respaldo válido.
## Reglas de Negocio
- RN-31: Solo personal autorizado puede ejecutar la recuperación de datos.
- RN-32: La recuperación debe utilizar únicamente respaldos verificados y completos.
## Requisitos No Funcionales
- RNF-004.2: La recuperación debe permitir restauración rápida minimizando la interrupción del negocio.
- RNF-004.3: Utilizar respaldo combinado: local y nube.
- RNF-002.2: La operación debe ejecutarse en tiempo de respuesta aceptable.
## Interfaces de Usuario
- Panel de administración de recuperación de datos.
- Alertas de estado de restauración.
- Registro de auditoría de acciones realizadas durante la recuperación.
## Referencias
- RNF-004.2: Recuperación de datos en tiempo limitado.
- RNF-004.3: Estrategia de respaldo combinada.
